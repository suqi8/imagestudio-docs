# 项目总览

## 项目定位

ImageStudio 是一个基于 Compose Multiplatform 的跨平台工具，核心目标是围绕 **ROM 定制与镜像处理工作流** 提供统一入口。

从现有仓库结构和说明来看，它主要承担以下职责：

- 管理 ROM 项目
- 进入项目工作台处理镜像相关任务
- 统一管理设置、更新、关于等公共页面
- 承载未来插件生态

## 平台支持

当前项目文档与构建脚本表明其目标平台包括：

- Android
- Windows
- macOS
- Linux

其中源码层面的主要目标是：

- `androidTarget`
- `jvm()`

这意味着 Android 和桌面端共享大量业务逻辑，再通过平台源集补足差异。

## 当前主要模块

根据源码目录与应用入口，主应用包括以下核心区域：

- **Main**：应用主容器与一级导航
- **Workshop**：项目列表、创建、删除、重命名
- **Workbench**：进入具体项目后的操作工作台
- **Plugin**：插件管理与插件执行入口
- **Settings**：设置中心
- **About / Update / Contributors**：关于、更新、贡献者等信息页

## 当前代码组织方式

主模块位于 `composeApp`，其中：

- `commonMain`：跨平台共享代码
- `androidMain`：Android 平台实现
- `jvmMain`：桌面平台实现
- `di`：依赖注入注册
- `feature`：按页面或功能域拆分
- `core`：工具能力、任务、设置等底层能力
- `plugin`：插件 API 与宿主侧实现

## 文档定位建议

如果这个文档站要长期维护，我建议把内容拆成三条主线：

- **用户线**：安装、使用、功能说明
